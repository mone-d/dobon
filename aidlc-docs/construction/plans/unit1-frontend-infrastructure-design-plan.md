# Infrastructure Design Plan - Unit 1 (Frontend)

## ユニット概要

**Unit 1: フロントエンド**
- 責務: ゲーム画面、ロビー・ルーム管理、プロフィール・統計画面の実装
- 技術スタック: Vue.js 3 + TypeScript + Pinia + Vite
- 実装方針: モックデータ、モックAPI、モックWebSocket を使用

---

## Infrastructure Design 計画

### [ ] 1. デプロイメント環境

#### 1.1 クラウドプロバイダーの選定

フロントエンドのホスティングに使用するクラウドプロバイダーについて、以下の質問にお答えください：

**Q1.1.1 クラウドプロバイダーの選択**

フロントエンドをホストするクラウドプロバイダーは？

A. Vercel
   - Vue.js/React に最適化
   - 自動デプロイ、自動スケーリング
   - 無料プラン利用可能

B. Netlify
   - Vue.js/React に最適化
   - 自動デプロイ、自動スケーリング
   - 無料プラン利用可能

C. AWS S3 + CloudFront
   - より細かい制御
   - コスト管理が必要

D. その他（Google Cloud, Azure 等）
   - 既存インフラとの統合

[Answer]:A

**Q1.1.2 環境構成**

デプロイメント環境をどのように構成しますか？

A. 本番環境のみ

B. 開発環境 + 本番環境
   - ステージング環境も含む

C. 開発環境 + ステージング環境 + 本番環境
   - 完全な環境分離

A. CDN（推奨）

B. クラウドストレージ + CDN

---

**Q3.1.2 キャッシング戦略**

CDN キャッシング戦略は？

A. 長期キャッシュ（推奨）

C. キャッシュなし

C. バックアップなし

### [ ] 4. メッセージング インフラストラクチャ

#### 4.1 リアルタイム通信

WebSocket によるリアルタイム通信のインフラについて、以下の質問にお答えください：

**Q4.1.1 WebSocket サーバー**

WebSocket サーバーをどこにホストしますか？

A. 専用サーバー（推奨）
   - Node.js + Socket.io サーバー
   - AWS EC2、Heroku 等

B. サーバーレス
   - AWS Lambda + API Gateway
   - より複雑

C. マネージドサービス
   - Firebase Realtime Database
   - より簡単

[Answer]:C
よくわからないのでコストが安くて簡単ならなんでも良いです。

---

**Q4.1.2 メッセージング方式**

メッセージング方式は？

A. WebSocket（推奨）
   - リアルタイム双方向通信
   - Socket.io で実装

B. Server-Sent Events (SSE)
   - サーバーからクライアントへの一方向通信

C. ポーリング
   - 定期的にサーバーをポーリング
   - 効率が低い

[Answer]:A

### [ ] 7. セキュリティ インフラストラクチャ

**Q7.1.1 認証方式**

ゲストユーザーの認証方式は？

A. ゲストID のみ（推奨）
   - シンプル、POC レベル

B. OAuth 2.0
   - Google、GitHub 等で認証

C. JWT トークン
   - より安全

[Answer]:A

---

**Q7.1.2 セッション管理**

セッション管理方式は？

A. ローカルストレージ + JWT
   - ステートレス

B. サーバーセッション
   - ステートフル

C. Cookie + HttpOnly
   - より安全

[Answer]:B
Aの方が簡単ならAでも良いです。

---

#### 7.2 データ保護

データ保護について、以下の質問にお答えください：

**Q7.2.1 データ暗号化**

データ暗号化は必要か？

A. 必要（本番環境）
   - HTTPS/WSS で通信暗号化
   - データベースの暗号化

B. 不要（POC レベル）
   - 暗号化なし

[Answer]:A
HTTPSはブラウザがHTTPは許可しなかったりとあるので対応したいですが、DBの暗号化などは不要です。

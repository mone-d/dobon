# テストシナリオ仕様書

## テスト環境
- URL: ngrok公開URL
- 認証: dobon / game2026
- プレイヤー: 2〜3人（スマホ複数ブラウザ）

---

## 1. ルーム管理

### 1.1 ルーム作成
- [ ] ロビー画面で「ルームを作成」→ 基本レート設定 → ルーム画面に遷移
- [ ] 4文字のルームコードが表示される
- [ ] プレイヤー数が「1/6」と表示される
- [ ] 作成者に「ゲーム開始」ボタンが表示される（無効状態）

### 1.2 ルーム参加
- [ ] 別ブラウザでルームコードを入力 → 「参加」→ ルーム画面に遷移
- [ ] 参加者側にルーム情報が表示される
- [ ] 作成者側にプレイヤー追加が反映される
- [ ] プレイヤー数が「2/6」に更新される
- [ ] 「ゲーム開始」ボタンが有効になる

### 1.3 ルーム退出
- [ ] 参加者が「ルームを退出」→ ロビーに戻る
- [ ] 作成者側でプレイヤーが減る

### 1.4 無効なルームコード
- [ ] 存在しないコードで参加 → エラーメッセージ表示

### 1.5 満員ルーム
- [ ] 6人参加後、7人目が参加 → エラーメッセージ表示

---

## 2. ゲーム開始

### 2.1 正常開始
- [ ] 作成者が「ゲーム開始」→ 全プレイヤーがゲーム画面に遷移
- [ ] 各プレイヤーに5枚の手札が配られる
- [ ] 場札が表示される（Aでない）
- [ ] ランダムに1人のターンが開始される
- [ ] 倍率が×1で表示される（初期Aがなければ）
- [ ] ターン順が表示される

### 2.2 初期Aによる倍率
- [ ] 初期場札決定時にAが出た場合、倍率が×2になる
- [ ] Aが連続した場合、倍率が累積する（×4, ×8...）

### 2.3 最小人数チェック
- [ ] 1人でゲーム開始 → エラー（最低2人必要）

---

## 3. 基本カード操作

### 3.1 カードプレイ（同じ数字）
- [ ] 場札と同じ数字のカードを選択 → 「1枚出す」ボタン表示 → 出せる
- [ ] 場札が更新される
- [ ] 手札が1枚減る
- [ ] ターンが次のプレイヤーに移る

### 3.2 カードプレイ（同じスート）
- [ ] 場札と同じスートのカードを選択 → 出せる
- [ ] 場札が更新される

### 3.3 カードプレイ（出せないカード）
- [ ] 場札と数字もスートも異なるカードを選択 → 「出す」ボタンが表示されない or 出せない

### 3.4 複数枚出し
- [ ] 同じ数字のカードを2枚以上選択 → 「X枚出す」ボタン表示
- [ ] 選択順序が数字バッジで表示される（①②③）
- [ ] 最初に選択したカードが一番下（場札と一致する必要あり）
- [ ] 最後に選択したカードが次の場札になる

### 3.5 異なる数字の選択
- [ ] 異なる数字のカードを選択 → 前の選択が自動解除される

### 3.6 カードドロー
- [ ] 自分のターンで山札をタップ → 手札が1枚増える
- [ ] 山札の残り枚数が減る
- [ ] ターンが次のプレイヤーに移る
- [ ] 自分のターンでない時は山札をタップしても何も起きない

### 3.7 ターン表示
- [ ] 自分のターン時に「🎯 あなたのターンです！」表示
- [ ] 自分のターン時に山札に緑の↑アイコン表示
- [ ] 他のプレイヤーのターン時は操作不可

---

## 4. 特殊カード: 8（ワイルド）

### 4.1 基本動作
- [ ] 8はどの場札に対しても出せる
- [ ] 8を出すとスート選択UIが表示される
- [ ] スートを選択するとバッジが場札に表示される
- [ ] 次のプレイヤーは選択されたスートのカードを出す必要がある

### 4.2 スート選択後のプレイ
- [ ] 選択されたスートと同じスートのカードが出せる
- [ ] 選択されたスートと異なるスートのカードは出せない（同じ数字8は出せる）

---

## 5. 特殊カード: A（スキップ）

### 5.1 基本動作
- [ ] Aを出すと次のプレイヤーがスキップされる
- [ ] スキップされたプレイヤーの次にターンが移る

### 5.2 複数枚出し
- [ ] Aを2枚出すと次の2人がスキップされる

---

## 6. 特殊カード: 9（リバース）

### 6.1 基本動作
- [ ] 9を出すとターン方向が逆転する
- [ ] ターン順表示に「← 逆」が表示される
- [ ] 逆方向でターンが進む

### 6.2 再度リバース
- [ ] もう一度9を出すと元の方向に戻る

---

## 7. 特殊カード: 2（強制ドロー）

### 7.1 被害者が2を持っていない場合
- [ ] 2が出される → 30秒カウントダウン表示
- [ ] 被害者に「X枚引く」ボタン表示
- [ ] 被害者がボタンを押す → 即座にX枚ドロー → ターン進行
- [ ] タイムアウト（30秒）→ 自動でX枚ドロー → ターン進行

### 7.2 被害者が2を持っている場合（押し付け）
- [ ] 2が出される → 30秒カウントダウン表示
- [ ] 被害者が手札から2を出す → 効果が次の人に移る（枚数スタック）
- [ ] 新しい被害者に30秒カウントダウン表示

### 7.3 スタッキング
- [ ] A→B(2)→C(2)→A: Aが4枚ドロー
- [ ] 枚数表示が正しく累積される（+2 → +4）

### 7.4 ドボンチャンス
- [ ] 2の猶予中にドボン宣言が可能
- [ ] ドボン成功 → 強制ドローは無効化

### 7.5 加害者・無関係者のUI
- [ ] カウントダウンバナーが表示される（「効果発動待ち...」）
- [ ] DOBONボタンは押せる

---

## 8. 特殊カード: K（手札公開）

### 8.1 被害者がKを持っていない場合
- [ ] Kが出される → 30秒カウントダウン表示
- [ ] 被害者に「オープンする」ボタン表示
- [ ] 被害者がボタンを押す → 手札が全員に公開される → ターン進行
- [ ] タイムアウト → 自動で手札公開 → ターン進行

### 8.2 被害者がKを持っている場合（押し付け）
- [ ] Kが出される → 30秒カウントダウン表示
- [ ] 被害者が手札からKを出す → 効果が次の人に移る
- [ ] 被害者自身は公開されない

### 8.3 公開状態の持続
- [ ] 公開されたカードは場に出されるまで公開状態
- [ ] 新たに引いたカードは非公開
- [ ] 公開カードを場に出すと、そのカードは手札から消える（自然に非公開に）

### 8.4 公開中のプレイヤーがKを出す
- [ ] 自分の公開状態に変化なし
- [ ] 次のプレイヤーにK効果が発動

### 8.5 ドボンチャンス
- [ ] Kの猶予中にドボン宣言が可能

---

## 9. ドボン宣言

### 9.1 ドボン成功
- [ ] 手札の全カードで四則演算（1種類のみ）→ 場札の数字と一致 → 成功
- [ ] 例: 手札[2,3], 場札5 → 2+3=5 → 成功
- [ ] 例: 手札[12,1], 場札11 → 12-1=11 → 成功
- [ ] 例: 手札[3,4], 場札12 → 3*4=12 → 成功
- [ ] 例: 手札[12,4], 場札3 → 12/4=3 → 成功

### 9.2 ドボン失敗（ペナルティ）
- [ ] 手札で場札の数字を作れない → 即座にゲーム終了
- [ ] 敗者（ドボン失敗者）が全員に支払い
- [ ] 支払い額: 山札から引いたカード × 基本レート × 倍率

### 9.3 自分ドボン禁止
- [ ] 自分が直前に出した場札に対してドボン → ルール違反 → ペナルティ

### 9.4 ターン外ドボン
- [ ] 自分のターンでなくてもDOBONボタンが押せる
- [ ] ドボン成功する

### 9.5 返しドボン
- [ ] Aがドボン宣言 → 10秒の返し猶予
- [ ] Bが返しドボン宣言 → Bが勝者
- [ ] 返しドボンの倍率: ×2 per counter

### 9.6 返し判定の非表示
- [ ] 各プレイヤーの返し宣言は他のプレイヤーには見えない
- [ ] 返しが発生した場合のみ全員に通知

### 9.7 複数人ドボン
- [ ] 最後に宣言したプレイヤーが勝者

---

## 10. 倍率システム

### 10.1 初期A
- [ ] ゲーム開始時にAが出た回数分 ×2

### 10.2 山札リシャッフル
- [ ] 山札が0枚になる → ドロー → リシャッフル発生 → 倍率×2
- [ ] 倍率表示が更新される

### 10.3 引きドボン
- [ ] 山札から引いたターンにドボン成功 → 倍率×2

### 10.4 オープンドボン
- [ ] 手札が全て公開状態でドボン成功 → 倍率×2

### 10.5 返しドボン
- [ ] 返し1回 → ×2、返し2回 → ×4

### 10.6 通常ドボン
- [ ] 通常ドボンでは倍率は加算されない（その時点の倍率を使用）

---

## 11. 支払い

### 11.1 ドボン成功時
- [ ] 敗者（場札を出した人）が勝者に支払い
- [ ] 支払い額: 山札から引いたカード × 基本レート × 倍率
- [ ] 支払いカードが結果画面に表示される

### 11.2 バースト時
- [ ] 手札14枚 → 即敗北
- [ ] 敗者が全員に支払い
- [ ] 支払い額: 山札から引いたカード × 基本レート × 倍率

### 11.3 ドボン失敗時
- [ ] バーストと同じ支払い

---

## 12. ゲーム終了・継続

### 12.1 ゲーム結果画面
- [ ] WIN/LOSE表示
- [ ] 支払い額表示
- [ ] 倍率表示
- [ ] 「もう一度プレイ」ボタン
- [ ] 「ロビーに戻る」ボタン

### 12.2 ゲーム継続
- [ ] 「もう一度プレイ」→ 同じメンバーで新しいゲーム開始
- [ ] 手札が再配布される
- [ ] 倍率がリセットされる

### 12.3 「次で抜ける」宣言
- [ ] ゲーム中に「次で抜ける」ボタン → 「退出予定」表示
- [ ] ゲーム終了後「もう一度プレイ」→ 全員ロビーに戻る

### 13.2 強制ドローによるバースト
- [ ] 2の効果で複数枚ドロー → 14枚以上 → バースト

### 13.3 バースト結果

- [ ] 敗者が全員に支払い

## 14. 接続・再接続

### 14.1 ブラウザ切り替え（ルーム待機中）
- [ ] A作成 → ブラウザ切替 → B参加 → ブラウザ切替 → Aにb参加が反映

### 14.2 ブラウザ切り替え（ゲーム中）
- [ ] ゲーム中にブラウザ切替 → 戻る → 最新のゲーム状態が表示される
- [ ] 切断中に相手が操作した内容が反映される

### 14.3 ゲーム開始の通知
- [ ] A作成 → ブラウザ切替 → B参加 → Bがゲーム開始 → ブラウザ切替 → Aがゲーム画面に遷移

### 14.4 長時間切断
- [ ] 30秒以上切断 → 戻った時に再接続・最新状態取得

## 15. ゲストID

### 15.1 新規ゲスト
- [ ] 名前入力 → 「ゲームを始める」→ ゲストID生成 → ロビーに遷移

### 15.2 既存IDで再開
- [ ] 「既存IDで再開」タブ → ID入力 → 「IDで再開」→ ロビーに遷移
- [ ] 前回のIDがlocalStorageから表示される

## 16. エッジケース

### 16.1 山札枯渇
- [ ] 山札0枚 + 捨て札あり → ドロー → リシャッフル → ドロー成功
- [ ] 山札0枚 + 捨て札なし → ドロー失敗（エラー表示なし）

### 16.2 全員パス状態
- [ ] 全員がドローしかできない状態 → ゲームは続行（山札が尽きるまで）

### 16.3 2人プレイでのリバース
- [ ] 2人プレイで9を出す → ターン方向が逆転するが、実質同じ（交互）

### 16.4 2人プレイでのスキップ
- [ ] 2人プレイでAを出す → 相手がスキップ → 自分のターンに戻る

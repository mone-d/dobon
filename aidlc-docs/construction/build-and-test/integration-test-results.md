## ブラウザテスト手順（Phase 2）

### 準備
1. **サーバーが起動していることを確認**
   - バックエンド: http://localhost:3000/ ✅
   - フロントエンド: http://localhost:5173/ ✅

### テスト手順

#### ステップ1: ホーム画面
1. ブラウザで http://localhost:5173/ を開く
2. 名前を入力（例: "テストユーザー"）
3. 「ゲームを始める」ボタンをクリック
4. ロビー画面に遷移することを確認

#### ステップ2: ロビー画面からRoom画面へ
1. 画面下部のデバッグボタン「Room」をクリック
2. Room画面に遷移することを確認
3. モックルーム情報が表示されることを確認
   - ルームコード: ABCD1234
   - プレイヤー数: 6/6
   - 基本レート: ¥100

#### ステップ3: ゲーム開始
1. 「ゲーム開始」ボタンをクリック
2. Game画面に自動遷移することを確認

#### ステップ4: コンソールログ確認
1. ブラウザの開発者ツール（F12）を開く
2. Consoleタブを確認
3. 以下のログが表示されることを確認:
   ```javascript
   Starting game: { roomId: "room-1", players: [...], baseRate: 100 }
   Game started successfully: "game-xxx"
   ```

#### ステップ5: バックエンドログ確認
1. ターミナルでバックエンドのログを確認
2. 以下のログが表示されることを確認:
   ```
   [INFO] Game start requested { roomId: "room-1", playerCount: 6, baseRate: 100 }
   [INFO] Game started successfully { roomId: "room-1", gameId: "game-xxx", playerCount: 6 }
   ```

#### ステップ6: ゲーム画面確認
1. プレイヤー情報が表示される（上部に対戦相手5人）
2. 場札が表示される（中央）
3. 自分の手札が表示される（下部）
4. 倍率が表示される（ヘッダー右側）
5. 現在のターンプレイヤーが表示される（ヘッダー中央）

### 期待される動作
- ✅ WebSocket接続が確立される
- ✅ `game:start` イベントが送信される
- ✅ バックエンドでゲームセッションが作成される
- ✅ `game:started` イベントが受信される
- ✅ `game:state-updated` イベントが受信される
- ✅ ゲーム画面に実際のゲーム状態が表示される（モックデータではなく）

### トラブルシューティング

#### WebSocket接続エラー
- **症状**: コンソールに "WebSocket connection failed" エラー
- **確認**: バックエンドサーバーが起動しているか確認
- **解決**: `cd backend && npm run dev` でバックエンドを再起動

#### イベント送信エラー
- **症状**: コンソールに "Socket not connected" 警告
- **確認**: `socketService.connect()` が呼ばれているか確認
- **解決**: ページをリロードしてWebSocket再接続

#### ゲーム状態が更新されない
- **症状**: ゲーム画面にモックデータが表示される
- **確認**: `game:state-updated` イベントが受信されているか確認
- **解決**: バックエンドログで `broadcastGameState` が呼ばれているか確認

#### プレイヤー情報が表示されない
- **症状**: ゲーム画面が空白
- **確認**: `gameState` が null でないか確認
- **解決**: `game:started` イベントが正しく受信されているか確認

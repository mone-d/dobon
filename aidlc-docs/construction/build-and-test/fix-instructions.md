# 修正指示書

<!-- ここに修正内容を記載してください -->

- 9を複数枚出したときはリバースが重なります。つまり、2枚なら正回転、3枚なら逆回転です。
- 3人プレイでAを出した時に正しくスキップされませんでした。
- 2人プレイでAを出した時には相手がスキップされるので、自分のターンに戻るはずですがそうなりませんでした。これはKや2でも同様です。
- A,9の効果が発生した時にEffectBannerは出ているのですが、消えるタイミングが次の人がカードを出した時になっていると思います。次の相手のターンが終わった時に消えるようにしてください。
- Kと2が発動した時に、EffectCountdownが勝手に消えてしまうことや出てこないことがあります。
- Kと2が発動している間にドボンが発生してゲームが終了し、次のゲームに移るとEffectCountdownが残っていることがあります。
- 1ゲーム終了したら次のゲームは前のゲームの敗者からスタートするようにしてください。
- この辺のEffectBannerとEffectCountdownの制御が微妙なので、どんな挙動をすべきか詳細に整理して一度教えてください。それをレビューします。
- 自分がドボンした後に結果待ちの間、自分がさらにドボンボタンを押せてしまいます。すると、チョボン扱いになってしまうので、さらに押せないように制御してください。
- ゲーム終了画面で、次のゲームへ行くボタンは全ユーザーが押せますが、誰か一人がおすと全員次のゲームに入ってしまうので、全員が押し終わったら次のゲームに進むようにしてください。なお、全員押してなくても15秒でタイムアウトとして次のゲームに進んで良いです。タイムアウトゲージはなくてよくて、次のゲームに進むボタンに 残りn秒 のように表示してくれれば良いです。
- 相手のオープンカードの見栄えについてです。相手のカード枚数によって見栄えが変わるので、画面のテスト（スクショなど活用）を網羅的に実施して見栄えをチェックしてください。
  - 1枚のカードの中のマーク3つが重なっている時があってなんのマークがわかりにくいです。少なくとも相手のカードが3枚の時に発現しました。
  - 多い枚数がオープンされている時にオープンされたカードが重なって、それぞれのマークと数字がわかりません。最大13枚オープンでもわかるようにして欲しいし、一部オープンの時はクローズカードとオープンカードの間の余白でさらに場所を使うことを考慮して画面レイアウトを実装してください。
- チョボン、バーストの時はその時のレートに追加で4倍して支払額計算をしてください。
- ドボン返しした時に、勝者も敗者も自分になってた気がします。昨日のテストプレイで見つけたのでログから調査できるかも。
- ゲームプレイ画面が見切れて、ドボンボタンが押しにくい端末があるようです。ゲームプレイ画面は必ずスクロールなしで表示できるようにしてください。
- 個人の統計情報で相手のが見られるのは良いですが、自分のもゲーム中に見られるようにしたいです。
- 長時間プレイして大体10回くらいリシャッフルが発生したのですが、途中で山札が0枚になってこれ以上引けない状態になってしまいました。手札を合計しても52枚には全然届きません。
- 支払い金額計算で残りの山札から1枚引くルールにしてますが、この数字がAだった場合はレートを倍にして、さらに1枚引いて金額計算するようにしてください。それがさらにAなら同様に倍を重ねていきます。
- ゲーム開始時の場札Aや支払い計算時のAが出てレートが倍になるタイミングは盛り上がりポイントですが、今はAが出たことがUI上わからず、レートだけ倍になっています。（支払いは未実装）UI上でもAが出たことがわかるようにして欲しいです。
- ロビー画面で参加ボタンが右にはみ出しています。
- 初期配布された手札で場のカードに対してドボンできる場合、天鳳です。他プレイヤー全員からレートの4倍掛けで賞金をもらえます。金額の決定は他のドボンなどと同じようにレートと山札から1枚引いたカードの掛け合わせです。またこの場合は敗者が一人に決まらないので次ゲームの開始は天鳳で上がった人としてください。

# Tech Stack Decisions - Unit 1 (Frontend)

## 概要

Unit 1（フロントエンド）の技術スタック選定理由を記載します。POC レベルのプロジェクトであり、チームメイトのみが利用するため、実績のある標準的なツールを選定しました。

---

## 1. フロントエンドフレームワーク

### 選定: Vue.js 3

**理由**:
1. **ユーザーの経験**: ユーザーが Vue.js に親しんでいる
2. **学習コスト**: 既知のフレームワークで開発効率が高い
3. **最新版**: Vue.js 3 は Composition API で最新の開発パターンに対応
4. **コミュニティ**: 活発なコミュニティと豊富なリソース

**代替案**: React
- React も選択肢として可能
- ユーザーが TypeScript に親しんでいれば React も検討可能
- 本プロジェクトでは Vue.js 3 を推奨

**決定**: **Vue.js 3** を採用

---

## 2. 言語

### 選定: TypeScript

**理由**:
1. **型安全性**: フロントエンドの複雑なロジックを型で保護
2. **開発効率**: IDE のインテリセンスで開発効率向上
3. **保守性**: 大規模化時のコード保守が容易
4. **ユーザーの経験**: ユーザーが TypeScript に親しんでいる

**設定**:
- TypeScript strict mode: 有効
- ESLint: 不要（POC レベル）
- Prettier: 不要（POC レベル）

**決定**: **TypeScript** を採用（strict mode 有効）

---

## 3. 状態管理

### 選定: Pinia

**理由**:
1. **Vue.js 3 との統合**: Vue.js 3 の公式推奨ライブラリ
2. **シンプルさ**: Redux より学習コストが低い
3. **型安全性**: TypeScript との相性が良い
4. **小規模プロジェクト**: POC レベルに適切

**代替案**: Redux（React の場合）
- React を選択した場合は Redux を使用
- Redux Toolkit で開発効率向上

**ストア構成**:
- ユーザーストア（currentUser, isLoggedIn）
- ゲームストア（gameState, gameHistory）
- ルームストア（rooms, currentRoom）
- 統計ストア（statistics, ranking）

**決定**: **Pinia** を採用

---

## 4. UI コンポーネントライブラリ

### 選定: 不要（カスタム実装）

**理由**:
1. **POC レベル**: シンプルな UI で十分
2. **依存最小化**: 外部ライブラリの依存を最小化
3. **カスタマイズ**: ゲーム特有の UI が必要
4. **バンドルサイズ**: 不要なコンポーネントを避ける

**実装方針**:
- Vue.js の基本コンポーネント機能を活用
- CSS は Tailwind CSS または SCSS で実装
- カード、ボタン等は自作

**決定**: **カスタム実装** を採用

---

## 5. スタイリング

### 選定: Tailwind CSS

**理由**:
1. **開発効率**: ユーティリティクラスで高速開発
2. **レスポンシブ**: モバイルファースト対応が容易
3. **カスタマイズ**: ゲーム特有のスタイルに対応
4. **バンドルサイズ**: 使用されないクラスを自動削除

**代替案**: SCSS
- より細かいスタイル制御が必要な場合
- Tailwind CSS と組み合わせて使用可能

**決定**: **Tailwind CSS** を採用

---

## 6. 通信ライブラリ

### 6.1 REST API 通信

**選定: Axios**

**理由**:
1. **標準的**: REST API 通信の事実上の標準
2. **シンプル**: 使いやすい API
3. **インターセプター**: リクエスト・レスポンス処理が容易
4. **エラーハンドリング**: 統一的なエラー処理

**設定**:
- ベース URL: 環境変数で切り替え
- タイムアウト: 10 秒
- リトライ: 最大 3 回

**決定**: **Axios** を採用

### 6.2 WebSocket 通信

**選定: Socket.io**

**理由**:
1. **自動再接続**: 接続切断時の自動再接続機能
2. **フォールバック**: WebSocket 非対応環境での対応
3. **イベント駆動**: ゲーム状態更新に適切
4. **実績**: 多くのプロジェクトで使用

**代替案**: 標準 WebSocket API
- より低レベルの制御が必要な場合
- Socket.io より軽量

**設定**:
- 自動再接続: 有効
- リトライ間隔: 1秒、2秒、3秒、4秒、5秒
- 最大リトライ回数: 5 回

**決定**: **Socket.io** を採用

---

## 7. ビルドツール

### 選定: Vite

**理由**:
1. **高速**: 開発サーバーの起動が高速
2. **ホットリロード**: ファイル変更時の即座な反映
3. **最適化**: 本番ビルドの最適化が優秀
4. **Vue.js 3 対応**: Vue.js 3 との完全統合

**代替案**: Webpack
- より細かい設定が必要な場合
- Vite で不足する場合の選択肢

**設定**:
- 開発サーバーポート: 5173
- ビルド出力: dist/
- ソースマップ: 開発環境で有効

**決定**: **Vite** を採用

---

## 8. テストフレームワーク

### 選定: Vitest（最小限）

**理由**:
1. **Vite 統合**: Vite との完全統合
2. **高速**: テスト実行が高速
3. **最小限**: POC レベルのため、テストは最小限

**テスト対象**:
- ビジネスロジック（ゲーム計算等）
- ストア（状態管理）
- ユーティリティ関数

**テスト非対象**:
- UI コンポーネント
- E2E テスト

**カバレッジ目標**: 30-50%（目標なし）

**決定**: **Vitest** を採用（最小限）

---

## 9. ブラウザ互換性

### 対応ブラウザ

**最新版のみ対応**:
- Chrome 最新版
- Safari 最新版
- Firefox 最新版
- Edge 最新版

**モバイルブラウザ**:
- iOS Safari 最新版
- Android Chrome 最新版

**ポリフィル**: 不要（ES2020 以上の機能を使用）

**決定**: **最新版のみ対応**

---

## 10. 開発環境

### 10.1 Node.js バージョン

**選定**: Node.js 18 LTS 以上

**理由**:
1. **LTS**: 長期サポート版で安定性が高い
2. **最新機能**: ES2020 以上の機能に対応
3. **パフォーマンス**: 最新版で高速

### 10.2 パッケージマネージャー

**選定**: npm または yarn

**理由**:
1. **標準**: npm は Node.js に付属
2. **互換性**: 両者とも互換性が高い
3. **ユーザーの選択**: ユーザーの好みで選択可能

**推奨**: npm（標準）

### 10.3 IDE

**推奨**: Visual Studio Code

**拡張機能**:
- Volar（Vue.js 3 サポート）
- ESLint（不要だが、あると便利）
- Prettier（不要だが、あると便利）

---

## 11. 本番環境

### 11.1 ホスティング

**選定**: 未定（ユーザーの選択に委ねる）

**候補**:
- Vercel（推奨、Vue.js/React 対応）
- Netlify（推奨、Vue.js/React 対応）
- AWS S3 + CloudFront
- Google Cloud Storage
- Azure Static Web Apps

**推奨**: Vercel または Netlify（簡単、無料）

### 11.2 通信

**HTTPS/WSS**: 必須

**SSL/TLS 証明書**:
- Vercel/Netlify: 自動で提供
- 自前ホスティング: Let's Encrypt 等で取得

### 11.3 環境変数

**設定**:
```
VITE_API_BASE_URL=https://api.example.com
VITE_WS_URL=wss://ws.example.com
```

**開発環境**:
```
VITE_API_BASE_URL=http://localhost:3000
VITE_WS_URL=ws://localhost:3000
```

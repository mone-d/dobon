# Business Rules - Unit 1 (Frontend)

## フロントエンドのビジネスルール

### 1. カード操作ルール

#### 1.1 カード出しの検証ルール

**ルール**: 出すカードは場札の条件を満たす必要がある

```
条件:
- 同じ数字のカード
- 同じスートのカード
- 特殊カード（8はワイルド）

検証:
- 出すカードが場札の条件を満たすか確認
- 満たさない場合はキャンセルで手札に戻る（警告メッセージは表示しない）
- ユーザーは再度カードを選択
```

#### 1.2 複数枚出しの検証ルール

**ルール**: 複数枚出す場合は同じ数字である必要がある

```
条件:
- 複数枚のカードが同じ数字

検証:
- 選択されたカードが全て同じ数字か確認
- 異なる数字が含まれている場合は前の選択を解除し、新しく選択されたカードを選択状態にする
- ユーザーは再度カードを選択
```

#### 1.3 山札から引くルール

**ルール**: 1ターンに1枚のみ山札から引ける

```
条件:
- 1ターンに1回のみ

検証:
- 既に山札から引いているか確認
- 引いている場合は「山札から引く」ボタンを無効化
- ターン終了後にボタンを有効化
```

### 2. ドボン宣言ルール

#### 2.1 ドボン宣言の検証ルール

**ルール**: 手札の全カードを使用し、四則演算のうち1種類のみで場札の数字を作る

```
条件:
- 手札の全カードを使用
- 四則演算のうち1種類のみ（+, -, *, /）
- 演算結果が場札の数字と一致

検証:
- 選択された演算式が有効か確認
- 手札の全カードを使用しているか確認
- 演算式が1種類のみか確認
- 演算結果が場札の数字と一致するか確認
- 検証失敗時は警告メッセージを表示
```

#### 2.2 ドボン不可ルール

**ルール**: 自分が直前に出したカードに対してはドボン不可（ただしドボン可能にすることもできるが、ルール違反で自動敗北）

```
条件:
- 自分が直前に出したカード

検証:
- 最後にカードを出したプレイヤーが自分か確認
- 自分の場合でもドボンは可能だが、ルール違反となり自動敗北
- 敗北時は支払いロジックがバースト時と同じ（敗者が全員に支払い）
- 他のプレイヤーの場合は通常のドボン
```

#### 2.3 ドボン宣言のタイミングルール

**ルール**: 自分の出したカードが場の最新でない時はいつでもドボン可能

```
条件:
- 自分の出したカードが場の最新でない

検証:
- 最後にカードを出したプレイヤーが自分でないか確認
- 自分でない場合はドボン可能
- 自分の場合はルール違反ドボンになる（自動敗北、バースト時と同じ支払い）
```

### 3. 返しドボン判定ルール

#### 3.1 返し宣言の非表示ルール

**ルール**: 返さない宣言は他のプレイヤーには見えない。返しが発生した場合は都度反映して他プレイヤーにもゲーム進行としてわかるようにする

```
条件:
- 返し判定フェーズ中

実装:
- 各プレイヤーの返し宣言UIは個別に表示
- 返さない宣言は他のプレイヤーに見えない
- 返しが発生した場合は WebSocket で game:return イベントを受信
- ゲーム画面に「返しドボン発生」と表示
- 返したプレイヤーの情報を表示
- 返し宣言は個別に送信
```

#### 3.2 返しドボン発生時の表示ルール

**ルール**: 返しが発生した場合は都度反映して他プレイヤーにもゲーム進行としてわかるようにする

```
条件:
- 返しドボンが発生

実装:
- WebSocket で game:return イベントを受信
- ゲーム画面に「返しドボン発生」と表示
- 返したプレイヤーの情報を表示
- 複数人の返しがある場合は最後に宣言したプレイヤーが勝者
```

#### 3.3 複数人ドボン時の勝者決定ルール

**ルール**: 1人目がドボン宣言、2人目以降は返しドボン宣言。複数人がドボン可能な場合、最後に宣言したプレイヤーが勝者

```
条件:
- 複数人がドボン宣言

実装:
- 1人目のドボン宣言を記録
- 2人目以降は返しドボン宣言として処理
- 各プレイヤーのドボン宣言時刻を記録
- 最後に宣言したプレイヤーを勝者として決定
- 支払い情報を表示
```

### 4. ゲーム継続ルール

#### 4.1 無限ゲーム継続ルール

**ルール**: 一度ゲーム開始後は無限に次のゲームが開始

```
条件:
- ゲーム終了後

実装:
- ゲーム終了後、自動的に次のゲームが開始
- プレイヤーが「次で抜ける」を宣言していない場合は継続
- プレイヤーが「次で抜ける」を宣言している場合は次のゲーム終了時に退出
```

#### 4.2 プレイヤー脱出ルール

**ルール**: ゲーム中に次で抜けることを宣言、一人抜けたら全員ルームに戻る

```
条件:
- プレイヤーが「次で抜ける」を宣言

実装:
- ゲーム中に「次で抜ける」ボタンを表示
- ユーザーが「次で抜ける」をクリック
- 次のゲーム終了時にプレイヤーが退出
- 一人でも退出した場合、全員ルームに戻る
- ルーム画面に遷移
```

### 5. ユーザー管理ルール

#### 5.1 ゲストログインルール

**ルール**: 新規ゲストIDを生成するか、既存のゲストIDを入力して再利用するかを選択

```
条件:
- ゲストログイン画面

実装:
- 「新規ゲスト」ボタン: 新規ゲストIDを生成
- 「既存ゲストID入力」ボタン: ゲストID入力フォームを表示
- ゲストIDはローカルストレージに保存するか、手入力で再利用するかを選択可能
- ユーザーが手動でゲストIDを入力することも可能
```

#### 5.2 ゲストID保存ルール

**ルール**: ゲストIDはブラウザのローカルストレージに保存するか、手入力で再利用するかを選択可能

```
条件:
- ゲストログイン後

実装:
- ゲストIDをローカルストレージに保存するか選択
- 次回アクセス時に自動的にゲストIDを読み込み（保存している場合）
- ユーザーが手動でゲストIDを入力して再利用することも可能
- ユーザーが手動でゲストIDを削除することも可能
```

#### 5.3 プロフィール編集ルール

**ルール**: ユーザー名、アバター、自己紹介を編集可能

```
条件:
- プロフィール編集画面

実装:
- ユーザー名: 1-20文字
- アバター: 画像ファイル選択
- 自己紹介: 0-100文字
- 「保存」ボタンをクリックしてサーバーに送信
```

---

### 6. 入力検証ルール

#### 6.1 ユーザー入力検証

**ユーザー名**
```
- 1-20文字
- 英数字、ひらがな、カタカナ、漢字、記号（-_）のみ
- 空白は許可しない
```

**基本レート**
```
- 正の整数
- 1以上の値
- 小数点は許可しない
```

**演算式**
```
- 有効な四則演算式
- 手札の全カードを使用
- 1種類の演算子のみ
```

**ゲストID**
```
- 英数字のみ
- 6-20文字
```

#### 6.2 カード操作検証

**出すカード**
```
- 場札の条件を満たす
- 同じ数字またはスート
- 複数枚出す場合は同じ数字
```

**複数枚選択**
```
- 同じ数字のみ
- 異なる数字は警告メッセージ
```

### 7. エラーハンドリングルール

#### 7.1 API エラー処理

**ネットワークエラー**
```
- リトライボタンを表示
- 最大3回まで自動リトライ
- 3回失敗後はユーザーに通知
```

**バリデーションエラー**
```
- エラーメッセージを表示
- ユーザーが修正して再試行
```

**サーバーエラー**
```
- エラーメッセージを表示
- ユーザーに通知
- リトライボタンを表示
```

#### 7.2 ユーザーエラー処理

**無効なカード操作**
```
- 警告メッセージを表示
- ユーザーが再度操作
```

**無効なドボン宣言**
```
- 警告メッセージを表示
- ユーザーが再度宣言
```

**無効なゲストID**
```
- エラーメッセージを表示
- ユーザーが再度入力
```

#### 7.3 接続エラー処理

**WebSocket 接続切断**
```
- 自動的に再接続を試みる
- 最大5回まで自動リトライ
- 5回失敗後はユーザーに通知
```

**再接続失敗**
```
- ユーザーに通知
- 手動で再接続するボタンを表示
```

### 8. 状態管理ルール

#### 8.1 グローバル状態の更新ルール

**ユーザーストア**
```
- ゲストログイン時に更新
- プロフィール編集時に更新
- ゲーム終了後に統計情報を更新
```

**ゲームストア**
```
- ゲーム開始時に初期化
- ターン進行時に更新
- ドボン宣言時に更新
- ゲーム終了時に結果を保存
```

**ルームストア**
```
- ルーム作成時に初期化
- プレイヤー参加・退出時に更新
- ゲーム開始時に更新
```

**統計ストア**
```
- ゲーム終了後に更新
- ゲーム履歴から計算
```

#### 8.2 ローカル状態の更新ルール

**GameBoard コンポーネント**
```
- selectedCards: カード選択時に更新
- isWaitingForResponse: API リクエスト送信時に更新
```

**DoboDeclarationUI コンポーネント**
```
- selectedFormula: 演算式選択時に更新
- formulaOptions: ゲーム状態変更時に更新
```

**ReturnDoboUI コンポーネント**
```
- isReturning: ユーザーが返す/返さないを選択時に更新
```

**CardHand コンポーネント**
```
- hoveredCard: マウスホバー時に更新
```

### 9. パフォーマンスルール

#### 9.1 レンダリング最適化ルール

```
- 不要な再レンダリングを避ける
- メモ化を使用して計算結果をキャッシュ
- 仮想スクロールを使用してリスト表示を最適化
- 画像の遅延読み込みを実装
```

#### 9.2 メモリ最適化ルール

```
- 不要なデータをメモリから削除
- 大きなデータ構造を避ける
- ガベージコレクションを活用
- メモリリークを防ぐ
```

#### 9.3 ネットワーク最適化ルール

```
- API リクエストをバッチ処理
- 不要なリクエストを避ける
- キャッシングを活用
- 画像の圧縮を実装
```

# Follow-up Questions - ドボンゲーム

## ゲームプレイに関する確認

### Q1: ドボン宣言のUI/UX
ドボン宣言時のユーザーインタラクションについて：

A) 手札から複数枚を選択して、演算式を入力する形式
B) 手札から複数枚を選択すると、システムが自動的に可能な演算式を提案する
C) 手札から複数枚を選択して、演算式を入力し、システムが検証する
D) その他

[Answer]: D
ドボン時は手札を全て使う必要があります。
そのため、演算式をユーザーが選択する形が良いです。
また、演算式を誤って選択してしまい、場のカードと一致しなかった場合はペナルティとしてバーストと同じ扱いの負けにしたいです。

### Q2: ドボン宣言のタイミング
ドボン宣言のタイミングについて：

A) 自分のターン中のみ宣言可能
B) 他のプレイヤーのターン中でも「返しドボン」として宣言可能
C) 両方対応したい

[Answer]: B
返しに限らず、自分の出したカードが場の最新でない時はいつでもドボン可能です。返しドボンは自分以外がドボンしたときはいつでも可能です。

# DOBON 統合テスト手順書

## 前提条件

- バックエンドが起動している（ポート3000）
- フロントエンドの依存関係がインストール済み

## テスト手順

### 1. フロントエンド起動

```bash
npm run dev
```

ブラウザで http://localhost:5173/ を開く

### 2. 基本フロー確認

#### 2.1 ゲストログイン
- [x] 名前を入力（例: "テストユーザー"）
- [x] 「ゲームを始める」ボタンをクリック
- [x] ロビー画面に遷移することを確認

#### 2.2 ルーム作成
- [x] 「ルームを作成」ボタンをクリック
- [x] 基本レートを設定（例: 100）
- [x] ルーム画面に遷移することを確認

#### 2.3 ゲーム開始
- [x] 「ゲーム開始」ボタンをクリック
- [x] ゲーム画面に遷移することを確認
- [x] 手札が表示されることを確認
- [x] 場札が表示されることを確認

### 3. WebSocket接続確認

ブラウザの開発者ツール（F12）を開き、コンソールを確認：

#### 3.1 接続ログ
```
✅ Socket connected
```

#### 3.2 ゲーム開始ログ
```
🎮 Starting game: { roomId, players, baseRate, myPlayerId }
✅ Game started successfully: <gameId>
📊 Game state updated: <gameState>
```

### 4. カード操作確認

#### 4.1 カード選択
- [x] 手札のカードをクリック
- [x] カードが選択状態になることを確認（緑の枠線）
- [x] 異なる数字のカードをクリック
- [X] 前の選択が解除されることを確認

#### 4.2 カードプレイ
- [x] カードを選択
- [x] 「カードを出す」ボタンをクリック
- [x] コンソールに以下のログが表示されることを確認：
```
🃏 Playing cards: { roomId, playerId, cards }
✅ Cards played successfully
📊 Game state updated: <gameState>
```

#### 4.3 カードドロー
- [x] 「カードを引く」ボタンをクリック
- [x] コンソールに以下のログが表示されることを確認：
```
🎴 Drawing card: { roomId, playerId }
✅ Card drawn: <card>
📊 Game state updated: <gameState>
```

### 5. エラーハンドリング確認

#### 5.1 接続エラー
- [ ] バックエンドを停止
- [ ] フロントエンドで操作を試行
- [ ] 画面上部にエラー通知が表示されることを確認
- [ ] エラーメッセージ: "サーバーに接続されていません"

#### 5.2 再接続
- [ ] バックエンドを再起動
- [ ] コンソールに以下のログが表示されることを確認：
```
🔄 Socket reconnected after <n> attempts
🔄 Rejoining game: { currentRoomId, currentUserId }
```

### 6. デバッグナビゲーション確認

画面右上のデバッグボタンで画面遷移を確認：
- [x] 「Lobby」ボタン → ロビー画面
- [x] 「Room」ボタン → ルーム画面
- [x] 「Game」ボタン → ゲーム画面

## 期待される結果

### 成功条件
- ✅ すべてのWebSocketイベントが正しく送受信される
- ✅ ゲーム状態がリアルタイムで更新される
- ✅ エラーが適切に処理され、通知される
- ✅ 再接続が自動的に行われる

### 失敗条件
- ❌ WebSocket接続エラーが発生する
- ❌ ゲーム状態が更新されない
- ❌ エラー通知が表示されない
- ❌ 再接続が失敗する

## トラブルシューティング

### WebSocket接続エラー

**症状**: `❌ Socket connection error`

**原因**:
1. バックエンドが起動していない
2. ポート番号が間違っている
3. CORSの設定が間違っている

**解決方法**:
```bash
# バックエンドの起動確認
cd ../backend
npm run dev

# ポート確認
lsof -i :3000

# 環境変数確認
cat .env.development
```

### ゲーム状態が更新されない

**症状**: カードをプレイしても画面が更新されない

**原因**:
1. `game:state-updated`イベントが受信されていない
2. gameStoreの状態更新が失敗している

**解決方法**:
```javascript
// ブラウザコンソールで確認
useGameStore.getState().gameState
useGameStore.getState().isConnected
```

### エラー通知が表示されない

**症状**: エラーが発生してもUIに表示されない

**原因**:
1. ErrorNotificationコンポーネントがレンダリングされていない
2. エラー状態が設定されていない

**解決方法**:
```javascript
// ブラウザコンソールで確認
useGameStore.getState().error
```
